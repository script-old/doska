
1) Пользователь через категории подает объявления
2) Каждый просмотр увеличивает счетчик просмотров на один или каждый УНИКАЛЬНЫЙ просмотр (уник через куки)
3) каждый логин пользователя считается для ведения статистики (и подключения НС в будущем)
4) каждое объявление имеет много  изображений (5-15)
5) для изображений вероятнее всего необходимо будет использовать carrierwave и mm-carrierwave , однако их необходимо тестировать. На данный момент будет простая модель Image с несколькими строками

6) Контроллеры - с использованием Sinatra
7) CSS? - MaterializeCSS || PureCSS || ?
8) Нужна модель страниц!
9) Необходим slug-генератор - https://github.com/rsl/stringex/tree/master
10) Отправка почты будет через Pony
11) Капча
12) Приватные данные - ruby dotenv
13) Пишем тесты и наращиваем мясо на модели
14) Для всех моделей необходимо написать свою валидацию, чтобы  использовать вывод ошибок на русском языке. Пример:
######################################
validate :custom_validation

  def custom_validation
    if age < 20
      errors.add( :age, "Youth is wasted on the young")
    end
  end
######################################
